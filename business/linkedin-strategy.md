### Post 8: Educational
```
I asked our team: "What's the one thing every developer should do before deploying a smart contract?"

The unanimous answer: write a threat model.

Here's a simple framework:

1. List all actors (users, admins, contracts, oracles)
2. Map all state-changing functions
3. For each function, ask: "What if a malicious actor calls this with unexpected inputs?"
4. Document trust assumptions
5. Review with someone who didn't write the code

This takes 2-4 hours and prevents 80% of critical vulnerabilities.

No tools needed. Just a document and critical thinking.

#smartcontract #solidity #development #security
```
